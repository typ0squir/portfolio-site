SNS 마케팅이 어려운 5060 시니어 자영업자를 위해, AI가 매출 및 날씨 분석 등을 통해 피드 이미지를 가공하고 맞춤형 인스타그램 캡션 게시물을 자동 생성 및 대리 업로드해 주는 마케팅 자동화 솔루션입니다.

  • Naver Map Playwright 크롤러와 Clova STT 음성 데이터를 활용한 가게 정보 자동 수집 및 온보딩 간소화
  • LangGraph 에이전트 및 ONNX 모델을 활용한 최적 프레임 추출과 AI 이미지 편집 파이프라인 구축
  • Resilience4j 기반 서킷 브레이커와 리트라이 패턴을 적용하여 다중 외부 API 연동 시 발생 가능한 연쇄 장애 전파 격리
  • Facebook Meta Graph API 연동으로 인스타그램 비즈니스 피드 자동 예약 및 즉시 발행 대행 구현
기존 웹 환경에 적응하기 어려운 5060 시니어 자영업자분들은 인스타그램 마케팅의 필요성을 느끼면서도, 복잡한 UI와 캡션 작성의 어려움으로 진입 장벽을 겪고 있습니다. 이를 해결하기 위해 말하듯 가게 정보를 입력하면 AI가 분석하여 자동으로 마케팅용 캡션과 고품질 피드 이미지를 제작하고 자동 업로드까지 대행해주는 통합 서비스인 '맡케팅'을 기획하게 되었습니다.

다량의 외부 리소스 API(가게 메뉴 정보 크롤링, 날씨 API, 생성 AI 서비스 등)를 수집하고 통합하는 환경이라 외부 서비스의 일시적인 네트워크 장애나 응답 지연이 서버 전체의 스레드 고갈 및 다운타임으로 이어질 위험이 있었습니다. 이를 해결하기 위해 Spring Boot 환경에서 외부 호출 API들에 대해 비동기 처리 및 서킷 브레이커를 적용하고, 예외 발생 시 안전한 백오프(Back-off) 정책을 동반한 재시도(Retry) 로직을 결합해 연쇄 장애 전파를 차단하고 견고한 내부 복구 프로세스를 안착시켰습니다.

이 과정에서 REST API의 신뢰성을 높이고 분산 환경에서 외부 장애에 흔들리지 않는 견고한 백엔드 안정성 설계 역량을 깊이 다질 수 있었습니다.

Spring Boot 3.5, Java 21, PostgreSQL, Redis, Resilience4j, FastAPI, Python 3.11, PyTorch, LangGraph, React 19, Vite, Zustand

#

대규모 오프라인 행사의 실시간 대기 혼잡도를 조율하고, 부스 도면 자동 픽셀 매핑 및 부스 관리자 가입 모니터링 시스템을 제공하는 통합 이벤트 관리 플랫폼입니다.

  • OpenCV를 활용한 2D 배치도 부스 영역 검출 처리 및 알고리즘 과적합 문제 분석
  • JWT 기반의 사용자 권한 다중 인가 계층 분리 및 행사 부스 관리자 대량 가입 프로세스 실시간 status 모니터링 시스템 구축
  • 엑셀 업로드를 활용한 부스 관리자 계정 일괄 생성 및 SMTP 기반 패스워드 설정 메일 발송 로직 구현
  • 상태 패턴(State Pattern)을 도입하여 부스 관리자의 실시간 승인 및 가입 단계를 체계적으로 관리
대규모 축제나 박람회에서는 부스별 대기 시간의 불균형과 혼잡도 제어가 어렵고 안전사고 우려가 빈번히 발생합니다. 주최 측에게는 배치도를 객체화하여 관리하기 쉬운 도구를 제공하고, 방문객에게는 모바일 원격 대기와 지도 매핑을 제공함으로써 혼잡을 선제적으로 조율하기 위해 시작되었습니다.

행사의 부스 도면 이미지로부터 부스 위치 영역을 검출해 인터랙티브 객체로 변환할 때, 특정 노이즈 필터링과 임계값이 특정 형태의 예시 도면에만 유독 맞춰져 검출율이 변동하는 '과적합(Overfitting) 현상'을 경험했습니다. 이를 극복하기 위해 아래와 같이 구체적인 3대 리팩토링 계획을 수립하고 고도화를 추진 중입니다.

  • 허프 변환(Hough Transform) 결합: 단순 Contour 검출에서 벗어나, 도면 내 수평/수직 벽면의 직선을 먼저 추출하여 부스 그리드 모델을 추정하는 하이브리드 벽면 구조 검출 구현.
  • YOLO 머신러닝 객체 검출 연동: 복잡한 도면 내 가구나 텍스트 노이즈에 구애받지 않도록 부스 영역 레이블 데이터를 축적해 경량 YOLO 모델 적용 및 오차 극복 테스트.
  • 수동 보정 인터랙티브 툴 도입: 검출 실패 시 주최자가 웹 UI 상에서 드래그하여 좌표를 손쉽게 수동 보정하고 동기화할 수 있는 관리자 화면 기획.

또한, 주최자가 대량의 부스 관리자 계정을 일괄 생성해 배포하는 가입 과정에서, 각 계정의 활성화 상태(status)를 실시간 모니터링할 수 있는 시스템을 구축해 주최자가 가입 현황에 따라 신속히 피드백하고 사후 후속 액션을 안내할 수 있도록 설계했습니다.

  • SSE(Server-Sent Events) 실시간 알림 서비스: 타인 피드의 좋아요/댓글 반응 및 실시간 저축 현황을 유저에게 전송하는 독립 알림 서버 추가 설계.
