RESOLUTION 해결 방안

API 호출부를 비동기 리액티브 웹클라이언트(WebClient)로 전환하고 개별 호출마다 Circuit Breaker를 적용했습니다. 특정 API 장애 발생 시 기본값(Fallback) 데이터를 반환하거나 재시도(Retry)하도록 우회 처리하여 연쇄 장애(Cascading Failure)를 차단했습니다.

오프라인 행사의 대기열 혼잡도를 제어하기 위해 모바일 대기열 관리, 부스 배치도 비전 객체화 분석 및 사후 통계 리포트 생성을 지원하는 통합 이벤트 관리 서비스입니다.

핵심 성과

  • OpenCV 이미지 처리를 통해 행사 기획자가 업로드한 2D 배치도 이미지 내 부스 공간을 검출(Contour Detection)하고 이 좌표 데이터를 가공하여 인터랙티브 지도 객체로 변환하는 기술 구현.
  • 행사 주최자(Host), 부스 관리자(Staff), 개인 사용자(Guest)로 구분된 3단계 JWT Role 기반 인가 처리 구조 설계.
  • 엑셀 업로드를 활용한 부스 관리자 계정 일괄 생성 및 SMTP 기반 패스워드 설정 메일 발송 로직 구현, 가입 진행 상태를 상태 패턴(State Pattern)으로 실시간 추적 및 대시보드 시각화.

ISSUE 문제 상황

복잡한 선과 텍스트가 섞인 손그림이나 복잡한 구조의 도면 이미지 업로드 시 부스 경계면(Contour) 검출 정확도가 현저히 떨어지거나 불필요한 노이즈가 부스로 파싱되는 문제 발생.

RESOLUTION 해결 방안

이미지 분석 전에 가우시안 블러(Gaussian Blur)와 적응형 임계처리(Adaptive Thresholding)를 적용하여 이미지 노이즈를 1차 제거하고, 특정 면적 이하 및 가로세로 비율(Aspect Ratio) 범위를 벗어나는 검출 객체는 필터링하도록 전처리 알고리즘을 고도화하여 인식 성공률을 92% 이상으로 끌어올렸습니다.

  • 관리자 웹에서 도서 카탈로그 수정, 불량 유저 및 신고 피드를 필터링할 수 있는 백오피스 어드민 기능과 권한 설정 제어 로직 구현.

ISSUE 문제 상황

도서 목록 조회 및 키워드 기반 복합 조건 필터링 쿼리 수행 시 데이터양이 10만 건을 넘어가면서 조회 속도가 1.5초 이상 지연되는 현상 발생.

RESOLUTION 해결 방안

도서 테이블의 자주 검색되는 복합 조건 컬럼(도서명, 저자, 출판일)에 복합 인덱스(Composite Index)를 구성하고, QueryDSL을 통한 커버링 인덱(Covering Index) 페이징 처리를 적용하여 쿼리 조회 레이턴시를 0.1초 내외로 단축했습니다.

  • 대학교 및 학과 대항전 형태로 실시간 저축 랭킹 데이터를 실시간 정렬하여 시각화해 주는 랭킹 시스템 개발.
  • 와이어프레임 기획과 캐릭터 성장 피드 화면의 직관적인 배치 등 게이미피케이션에 최적화된 사용자 경험(UX) 중심 화면 레이아웃 설계.

ISSUE 문제 상황

학교/학과 랭킹 대항전 등 실시간 납입 건수가 일시에 몰리면서 매번 랭킹 집계를 위해 랭킹 조회 API 호출 시 DB 집계(Aggregate) 함수(SUM, COUNT)가 연쇄 호출되어 API 응답이 극도로 지연되는 현상 발생.

RESOLUTION 해결 방안

Redis의 Sorted Set(ZSET) 자료구조를 적용하여 랭킹 점수(누적 저축 금액)를 메모리 상에서 실시간으로 정렬 및 업데이트하도록 개선하였습니다. 이를 통해 랭킹 조회 레이턴시를 5ms 미만으로 감소시켰고 DB 경합 현상을 완벽히 해결했습니다.
